# Project Planning

## Purpose
Turn an idea into an executable Effort.

## Inputs
- Project idea
- Desired outcome
- Constraints
- Deadline
- Existing notes or resources

## Workflow
1. Define the outcome.
2. Clarify why it matters.
3. Identify scope and non-scope.
4. Break work into milestones.
5. Define next actions.
6. Identify risks and dependencies.
7. Link supporting Atlas notes and Sources.

## Output Format
```markdown
# Project - Name

## Outcome

## Why It Matters

## Scope
- 

## Milestones
- 

## Next Actions
- [ ] 

## Risks
- 

## Related
- 
```

## Rules
- Store active projects in `Efforts/Active`.
- Keep the first version simple.
- Make the next action obvious.

## Example
Input: "Build HalalHub."
Output: Project note with outcome, milestones, next actions, and risks.
